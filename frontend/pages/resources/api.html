Developer Platform

Integrate FeedBack Analyzer into your feedback stack

The API is maintained by the platform engineering pod in Lucknow. Every endpoint uses OAuth 2.0 client credentials, scoped keys, and signed webhooks.

Workflow

Queue an analysis

Upload files (CSV, JSON, TXT) to object storage, then create an analysis request. Poll the job or subscribe to callback events.

  • Batch size: up to 5,000 comments per request
  • Retry strategy: exponential backoff (up to 5 attempts)
  • Webhook signatures: HMAC-SHA256 with workspace secret
POST /api/v1/analyses
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "Voice of Rider - Lucknow Metro",
  "language": "multilingual",
  "files": [
    "s3://feedback-analyzer/uploads/lucknow_riders_2025.csv"
  ],
  "callbacks": [
    "https://ops.example.com/hooks/feedback"
  ]
}

Authentication

Create client credentials from the dashboard. Secrets are encrypted with AWS KMS and rotated automatically every 90 days.

  • Scopes: ingestion, analyses:read, exports, admin
  • IP allow lists optional for regulated deployments
  • Audit logs capture every token exchange in IST

Webhooks

Subscribe to `analysis.completed`, `analysis.failed`, and `export.ready`. Each payload includes HMAC headers and replay protection.

  • Delivery timeout: 10 seconds per attempt
  • Automatic suppression after 24 hours of failures
  • Verification CLI available for local testing

SDKs

Official TypeScript and Python SDKs are published from our Lucknow CI pipeline. Each release is signed and mapped to the OpenAPI spec.

  • Package names: @feedback-analyzer/sdk and feedback-analyzer
  • Semantic versioning with weekly canary drops
  • Examples: ingestion queues, alerts, governance checks
